# Help Command

Status: Implemented

## Overview

The `help` command displays usage information for Ralph CLI and its subcommands.

## Purpose

- Provide users with contextual help about available commands and flags
- Support discovery of CLI functionality through examples
- Enable self-service documentation without external resources

## Command Interface

```bash
# Show root help (same as `ralph --help`)
ralph help

# Show command-specific help
ralph help <command>
```

### Flags

None. The command accepts no flags.

### Examples

**Root help:**
```bash
$ ralph help
Ralphex is a POSIX-compliant AI agentic loop shell runner.

Usage:
  ralph [options] [prompt] [scope] [flags]
  ralph [command]

Examples:
  ralph build
  ralph plan my-feature
  ralph --max-iterations 10 build

Available Commands:
  completion  Generate the autocompletion script for the specified shell
  help        Help about any command
  init        Initialize Ralphex configuration
  prompts     List available prompts
  run         Run a prompt loop
  version     Print the version number of Ralph CLI

Flags:
  -a, --agent string    AI agent to use (default: opencode)
  -c, --config string   Config file to source
  -h, --help            help for ralph
  -m, --max-iterations  Maximum iterations (default: 25)
  ...

Use "ralph [command] --help" for more information about a command.
```

**Command-specific help:**
```bash
$ ralph help init
Initialize Ralphex configuration by creating a toml config file.

Usage:
  ralph init [flags]

Flags:
  -f, --force     Overwrite existing config file
  -h, --help      help for init
  -i, --interactive  Run in interactive mode (default: true)

Examples:
  ralph init              # interactive setup
  ralph init --force      # overwrite existing config
  ralph init --no-interactive  # use defaults
```

## UX Decisions

### Root help shows:

1. **Brief description** — One paragraph explaining what Ralphex does
2. **Usage pattern** — Shows both flag-based and command-based invocation
3. **Examples** — 3-5 common invocations (build, plan, custom prompt)
4. **Available commands** — Auto-discovered from registered subcommands
5. **Flags** — All global flags with descriptions and defaults
6. **Call-to-action** — "Use `ralph [command] --help` for more information"

### Command-specific help shows:

1. **Command description** — One-line purpose statement
2. **Usage pattern** — Command-specific syntax
3. **Command flags** — Only flags relevant to that command
4. **Examples** — If applicable, 2-3 common invocations for that command

### Help discovery:

- `ralph help` and `ralph --help` produce identical output
- `ralph help <command>` and `ralph <command> --help` produce identical output
- Unknown commands return error: `unknown command "<name>" for "ralph"`

## Implementation Details

### Cobra integration

Help is provided automatically by the Cobra framework. No custom implementation required.

**Registration:**
```go
// Automatic - Cobra registers help by default
cmd := &cobra.Command{
    Use:   "ralph",
    Short: "POSIX-compliant AI Agentic Loop runner",
    // ...
}
```

**Custom help template (optional):**
```go
// If we want to customize help format in the future:
cmd.SetHelpTemplate(customTemplate)
```

### Completion command

The `completion` command (auto-generated by Cobra) is also listed in help output:
```bash
$ ralph help completion
Generate the autocompletion script for the specified shell

Usage:
  ralph completion [bash|zsh|fish|powershell]

Examples:
  source <(ralph completion bash)
  ralph completion zsh > ~/.zsh/_ralph
```

## No custom implementation needed

Help is fully provided by Cobra framework. Custom implementation only required if:
- We want to customize help output format (fonts, colors, layout)
- We want to add dynamic examples based on project state
- We want contextual help (e.g., "you're missing specs, run `ralph init` first")
- We want to add tutorials or guided workflows

## Output Format

Help output is plain text with:
- Sections separated by blank lines
- Commands aligned in columns
- Flags with descriptions on same line (if fits) or wrapped
- Examples indented

## Exit Codes

- `0`: Help displayed successfully
- `1`: Unknown command requested (e.g., `ralph help nonexistent`)

## Testing

Tests verify:
- Help command executes without error
- Output contains expected sections (Usage, Available Commands, Flags)
- Command-specific help works for each subcommand
- Unknown commands return appropriate error

Test location: `internal/cli/cmd_test.go`

Run tests with:
```bash
make test
```

## Security Considerations

- Help output is public and non-sensitive
- No secrets or credentials exposed
- Examples use safe, non-production values
